什么是 OpenClaude？

AI 编程助手 —— 但有一个巧妙的设计

Claude Code 是一个命令行 AI 编程助手。OpenClaude 是它的一个分支 —— 意思是有人拿了原始代码，在上面加了一个重要的想法：

不仅能与 Claude（Anthropic 的 AI）合作，OpenClaude 还能与任何 AI 配合工作。

GPT-4o、DeepSeek、Gemini、Ollama（在你的电脑上运行）、Mistral、Groq……全部支持。

可以这样理解：原始的 Claude Code 就像一个高端音响系统，只能接受一个品牌的磁带。OpenClaude 换掉了磁带卡座，换成了一个通用的 适配器 ，可以接受任何格式的磁带。

导演：QueryEngine

QueryEngine 是整个代码库中最重要的文件。它位于 src/QueryEngine.ts，大小为 46 KB——它最大是有原因的。

它的职责是：接收你的消息，调用 AI，检查 AI 是否想使用工具，运行工具，将结果反馈给 AI，然后重复——直到 AI 说"我说完了"。

CODE 

async *submitMessage(
  prompt: string,
): AsyncGenerator<SDKMessage> {
  // add user message to history
  this.messages.push(userMsg);
  // call AI, loop until done
  while (true) {
    const response = await api.call(messages);
    if (response.stop_reason === 'end_turn') break;
    await this.executeTools(response);
  }
}
通俗解释

一个随时间产生消息的函数（* 表示它是一个生成器——像一个工厂，持续生产直到被叫停）。

它接收你输入的提示词作为输入。

SDKMessage 是它产生的对象类型——对话的构建块。

将你的消息添加到对话的运行历史中。

无限循环——等准备好了就退出。

将目前所有的消息发送给 AI 并等待响应。

如果 AI 说"我说完了"，退出循环。

否则，AI 想使用工具——运行那些工具然后再循环。

03

魔法翻译器

OpenClaude 如何在 10+ 个不同的 AI 提供商之间切换而不需要为每个提供商重写代码？

语言障碍问题

每家 AI 公司都发明了自己的通信方式。Anthropic 的 API 格式与 OpenAI 的不同。如果 OpenClaude 在整个代码库中使用 Anthropic 的格式，它就永远无法与 GPT-4o 通信。

🤖
Anthropic 格式 { type: "tool_use", id: "...", name: "...", input: {} }
≠
🤖
OpenAI 格式 { tool_calls: [{ id: "...", function: { name: "...", arguments: "{}" } }] }

解决方案是一种叫做 适配器 的设计模式。OpenClaude 在内部使用一个——应用程序的其余部分都使用 Anthropic 的语言，适配器默默地将其转换为实际提供商需要的格式，反之亦然。

认识 openaiShim.ts — OpenClaude 的核心，1,098 行代码

文件 src/services/api/openaiShim.ts 是 OpenClaude 相对于原始 Claude Code 做出的最重要改进。它做三件事：

1
转换发出的消息

接收 QueryEngine 发出的 Anthropic 格式消息，在发送给 AI 前将其重写为 OpenAI 格式。

2
转换收到的数据流

读取 OpenAI 格式的流式响应，将每个数据块转换回 QueryEngine 能理解的 Anthropic 格式。

3
修复损坏的 JSON

当 AI 模型在词中间流式传输工具参数时，JSON 是不完整的。shim 通过尝试常见结尾直到某个能正确解析来自动补全。

// src/services/api/openaiShim.ts — 简化概念

你的提示词 → Anthropic 格式 → openaiShim 转换 → OpenAI 格式 → GPT-4o
GPT-4o 响应 → OpenAI SSE 流 → openaiShim 转换 → Anthropic 事件 → 你的终端

聪明的 JSON 修复技巧

当 AI 模型实时响应时（叫做 流式传输）， 它们有时会在句子中间中断——特别是在 JSON 这样的结构化数据中。当 AI 正在描述一个工具调用时，这是一个真正的问题。

代码 

const combinations = [
  '}', '"}' , ']}' ,
  '"]}' , '}}' , '"}}' 
];
for (const combo of combinations) {
  try {
    JSON.parse(str + combo);
    suffixToAdd = combo;
    break;
  } catch {}
}
通俗解释

JSON 字符串可能被截断的最常见方式的列表——只是一个右花括号，或者右引号+花括号，等等。

一个一个地尝试每个可能的结尾。

尝试用这个后缀附加到损坏的 JSON 上来解析……

如果解析成功！把这个后缀记为修复方案。

停止尝试——我们找到了正确的结尾。

如果解析失败，静默忽略错误并尝试下一个后缀。

💡 这为什么对你重要

如果你在使用非 Claude 模型时看到 OpenClaude 因"JSON 解析错误"而失败，这很可能是一个流式传输问题。shim 已经尝试修复它——但存在边缘情况。了解这一点后，你可以告诉你的 AI 助手："openaiShim.ts 的 combinations 数组需要一个新的后缀来处理这种响应格式。"

流式传输：为什么你看到文字逐字出现

AI 不是先写完完整答案再发送。它逐个 token 发送——就像电报操作员一次一个字母地敲出消息。shim 必须拦截这个实时流，转换每个数据块，然后传递下去。

流式响应是如何到达的：

数据块 1: { "delta": { "text": "The" } }
数据块 2: { "delta": { "text": " answer" } }
数据块 3: { "delta": { "text": " is..." } }
最终: { "stop_reason": "end_turn" }

shim 实时将每个 OpenAI 格式的数据块转换为 Anthropic 格式的事件。QueryEngine 永远不会知道它在与 Claude 以外的东西对话。

权限系统：每个操作的安全开关

在任何工具运行之前，OpenClaude 都会问：这允许吗？这叫做 权限系统—— 这就是 OpenClaude 即使在 AI 决定要运行 shell 命令时也能保持安全的方式。

代码 

const wrappedCanUseTool = 
  async (tool, input) => {
  const result = await canUseTool(
    tool, input
  );
  if (result.behavior !== 'allow') {
    this.permissionDenials
      .push(tool.name);
  }
  return result;
};
通俗解释

创建一个包装每次工具调用的守门员函数。

它接收工具名称和其建议的输入作为参数。

询问权限系统：这个工具调用可以吗？

传递正在调用的工具及其数据。

结果告诉我们行为：允许、拒绝或询问用户。

如果该工具未被允许……

记录哪个工具被拒绝。

（此日志帮助 AI 理解为什么它不能做某事。）

将决定返回给调用者。

🛡️ 为什么这很重要

权限系统是将"编程助手"和"可以对你的电脑做任何事的 AI"区分开来的关键。你保持控制权。当你告诉 AI 助手"自动运行命令"时——你正在禁用这个系统。要知道什么时候合适，什么时候不合适。

当出现问题时

每个 AI 工具最终都会出现问题。构建心智模型来跳出 bug 循环。

AI bug 循环——以及如何跳出它

你可能经历过：你让 AI 修复一个 bug，它做了修改，bug 仍然存在，你再次询问，它又做了另一个修改，同样的 bug 仍然存在……这叫做 bug 循环。

🔴

Bug 循环的迹象

AI 以微小变化尝试相同方法 2-3 次。它已经没有想法但不会说出来。

✅

如何跳出

停下来给 AI 新信息。自己阅读错误信息。粘贴确切错误。转向："不要修复它——解释为什么会发生这种情况。"

💡 关键洞察

OpenClaude（以及一般的 AI 工具）只能对上下文窗口中的内容进行推理。如果真正的问题不在对话中，AI 实际上看不到它。你必须把缺失的信息带进来。

按层诊断：首先检查哪里

有了本课程的建筑理解，你可以将任何问题缩小到特定层。以下是诊断图：

完全没有响应 → 检查 API 层：基础 URL 错误、API 密钥缺失或网络问题
工具调用乱码 → 检查 openaiShim.ts：提供商可能有 shim 未处理的格式怪癖
任务中途冻结 → 检查 QueryEngine：工具执行循环可能正在等待你错过的权限提示
工具运行但失败 → 检查工具系统：工具可能缺乏权限（文件系统、网络等）或输入格式错误
输出格式错误 → 模型本身可能是问题所在：尝试使用不同的提供商或更强的模型使用相同提示词

切换提供商：独特的调试工具

大多数 AI 工具将你锁定在一个模型上。OpenClaude 不是。这不仅仅是一个功能——它是一种诊断能力。

🔍
隔离模型 bug 和工具 bug

如果相同提示词在 Claude 上有效但在 GPT-4o 上失败，bug 在 shim 或 GPT-4o 的行为中——不在你的提示词中。如果两者都失败，问题在其他地方。

💰
低成本复现，用最佳模型修复

使用廉价快速的模型（DeepSeek、Groq）来复现和迭代调试。只有当你有明确的修复方案要验证时，才切换到 Claude Opus。

🏠
API 宕机时切换离线

如果 Anthropic 或 OpenAI 出现宕机，立即切换到 Ollama。你的工作流程不会中断。

OpenClaude 如何处理不稳定的 AI 响应

调用 AI API 时， 速率限制 和临时网络故障很常见。OpenClaude 有一个专门的文件来处理这些情况：src/services/api/withRetry.ts。

代码 

async function withRetry(fn, opts) {
  let attempt = 0;
  while (attempt < opts.maxRetries) {
    try {
      return await fn();
    } catch (err) {
      if (isRateLimit(err)) {
        await sleep(backoff(attempt));
      } else throw err;
    }
    attempt++;
  }
}
通俗解释

一个可以自动重试任何操作的包装函数。

将计数器从零开始。

持续尝试直到达到最大重试次数。

尝试运行该操作。

如果成功了，立即返回结果——不需要更多重试。

如果失败了，检查它是什么类型的错误。

如果是速率限制错误，在重试前等待（每次等待时间更长——指数退避）。

对于任何其他错误，放弃并重新抛出它——重试不会有用。
