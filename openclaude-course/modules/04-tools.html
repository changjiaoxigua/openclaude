AI 如何完成任务

AI 模型只能写文字。那么 OpenClaude 是如何运行命令、编辑文件和搜索网页的？认识一下工具系统。

AI 是一个被锁在房间里的聪明大脑

语言模型本身只能生成文字。它不能打开文件、运行命令或访问互联网。但 OpenClaude 在墙上给它提供了一组标签按钮——当它按下按钮时外部世界就会执行操作并报告结果。

权限系统：每个操作的安全开关

在任何工具运行之前，OpenClaude 都会问：这允许吗？这叫做 权限系统—— 这就是 OpenClaude 即使在 AI 决定要运行 shell 命令时也能保持安全的方式。

代码 

const wrappedCanUseTool = 
  async (tool, input) => {
  const result = await canUseTool(
    tool, input
  );
  if (result.behavior !== 'allow') {
    this.permissionDenials
      .push(tool.name);
  }
  return result;
};
通俗解释

创建一个包装每次工具调用的守门员函数。

它接收工具名称和其建议的输入作为参数。

询问权限系统：这个工具调用可以吗？

传递正在调用的工具及其数据。

结果告诉我们行为：允许、拒绝或询问用户。

如果该工具未被允许……

记录哪个工具被拒绝。

（此日志帮助 AI 理解为什么它不能做某事。）

将决定返回给调用者。

🛡️ 为什么这很重要

权限系统是将"编程助手"和"可以对你的电脑做任何事的 AI"区分开来的关键。你保持控制权。当你告诉 AI 助手"自动运行命令"时——你正在禁用这个系统。要知道什么时候合适，什么时候不合适。

生成工具的工具：AgentTool

工具箱中最强大的工具是 AgentTool。它允许主 AI 生成一个 子代理—— 一个完全独立的 AI 会话——来处理特定的子任务。子代理有自己的上下文、工具和对话历史。

主代理

"迁移数据库架构并更新所有使用旧架构的 API 端点。"

上下文太大无法处理？生成子代理。

→
子代理 1

只处理数据库迁移

子代理 2

更新 API 端点以使用新架构

💡 高阶操作：在你的提示词中使用它

你可以明确要求 OpenClaude 使用子代理："使用子代理来研究代码库，然后用你的主上下文来编写实现。"这可以保持主对话的整洁和专注。
