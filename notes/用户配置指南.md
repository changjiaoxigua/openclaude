# 用户配置指南：如何配置 OpenAI API Key

## 📋 概述

当用户安装打包后的 OpenClaude 后，有多种方式配置 OpenAI API Key。本文档详细说明所有配置方法。

---

## 🎯 配置方式一览

| 配置方式 | 难度 | 持久性 | 适用场景 |
|---------|------|--------|---------|
| 环境变量（临时） | ⭐ | 临时 | 快速测试 |
| Shell 配置文件 | ⭐⭐ | 永久 | 长期使用 |
| Profile 文件 | ⭐⭐⭐ | 永久 | 多 Provider 切换 |
| 命令行参数 | ⭐ | 临时 | 单次使用 |
| 交互式配置 | ⭐⭐ | 永久 | 初次使用 |

---

## 方式一：环境变量（最简单）

### Windows PowerShell

```powershell
# 设置环境变量
$env:CLAUDE_CODE_USE_OPENAI="1"
$env:OPENAI_API_KEY="sk-your-api-key-here"
$env:OPENAI_MODEL="gpt-4o"

# 启动 OpenClaude
openclaude
```

### macOS / Linux

```bash
# 设置环境变量
export CLAUDE_CODE_USE_OPENAI=1
export OPENAI_API_KEY=sk-your-api-key-here
export OPENAI_MODEL=gpt-4o

# 启动 OpenClaude
openclaude
```

**优点**：
- ✅ 最简单直接
- ✅ 无需编辑文件
- ✅ 立即生效

**缺点**：
- ❌ 关闭终端后失效
- ❌ 每次都需要重新设置

---

## 方式二：Shell 配置文件（推荐）

### macOS / Linux

编辑 `~/.zshrc` 或 `~/.bashrc`：

```bash
# 打开配置文件
nano ~/.zshrc

# 添加以下内容
export CLAUDE_CODE_USE_OPENAI=1
export OPENAI_API_KEY=sk-your-api-key-here
export OPENAI_MODEL=gpt-4o

# 保存并重新加载
source ~/.zshrc
```

### Windows

**方法 1：系统环境变量**

1. 右键"此电脑" → "属性"
2. "高级系统设置" → "环境变量"
3. 在"用户变量"中添加：
   - `CLAUDE_CODE_USE_OPENAI` = `1`
   - `OPENAI_API_KEY` = `sk-your-api-key-here`
   - `OPENAI_MODEL` = `gpt-4o`

**方法 2：PowerShell Profile**

```powershell
# 打开 PowerShell Profile
notepad $PROFILE

# 添加以下内容
$env:CLAUDE_CODE_USE_OPENAI="1"
$env:OPENAI_API_KEY="sk-your-api-key-here"
$env:OPENAI_MODEL="gpt-4o"
```

**优点**：
- ✅ 永久保存
- ✅ 每次启动自动加载
- ✅ 一次配置，长期使用

**缺点**：
- ❌ 需要编辑配置文件
- ❌ 需要重新加载配置

---

## 方式三：Profile 文件（高级）

OpenClaude 支持 **Profile 系统**，可以在项目目录下创建 `.openclaude-profile.json` 文件。

### 创建 Profile 文件

在项目根目录创建 `.openclaude-profile.json`：

```json
{
  "profile": "openai",
  "env": {
    "OPENAI_API_KEY": "sk-your-api-key-here",
    "OPENAI_MODEL": "gpt-4o",
    "OPENAI_BASE_URL": "https://api.openai.com/v1"
  },
  "createdAt": "2025-01-01T00:00:00.000Z"
}
```

### 使用 Profile 启动

```bash
# 方式 1：直接启动（自动加载当前目录的 Profile）
openclaude

# 方式 2：使用 Profile 启动脚本
bun run profile:init -- --provider openai --model gpt-4o
```

### Profile 文件位置

```
~/.openclaude-profile.json          # 全局 Profile（家目录）
/path/to/project/.openclaude-profile.json  # 项目 Profile
```

**优先级**：项目 Profile > 全局 Profile > 环境变量

**优点**：
- ✅ 支持多个 Profile
- ✅ 可以在不同项目使用不同配置
- ✅ 配置文件可以共享

**缺点**：
- ❌ 需要手动创建 JSON 文件
- ❌ API Key 明文存储（注意安全）

---

## 方式四：命令行参数

直接在命令行传递 API Key：

```bash
# 使用环境变量方式
CLAUDE_CODE_USE_OPENAI=1 OPENAI_API_KEY=sk-your-key openclaude

# 或使用 bun 脚本
bun run dev:openai
```

**优点**：
- ✅ 无需配置文件
- ✅ 适合临时使用

**缺点**：
- ❌ 每次都需要输入
- ❌ API Key 会出现在命令历史中

---

## 方式五：交互式配置（推荐新手）

OpenClaude 提供了交互式配置脚本。

### 使用配置脚本

```bash
# 自动检测并配置（推荐）
bun run profile:auto

# 手动选择 Provider
bun run profile:init -- --provider openai --model gpt-4o

# 配置 Ollama 本地模型
bun run profile:fast

# 配置代码模型
bun run profile:code
```

### 配置流程

```
$ bun run profile:init -- --provider openai --model gpt-4o

✔ Creating OpenAI profile...
✔ Model: gpt-4o
✔ API Key: [请输入你的 API Key]
✔ Profile saved to: .openclaude-profile.json
✔ Ready to use!
```

**优点**：
- ✅ 引导式配置
- ✅ 自动验证配置
- ✅ 适合新手

**缺点**：
- ❌ 需要安装 Bun
- ❌ 需要源码或开发环境

## 🔐 安全最佳实践

### 1. 不要提交 API Key 到 Git

在 `.gitignore` 中添加：

```gitignore
# API Keys
.env
.openclaude-profile.json

# Secrets
*_api_key.txt
secrets.json
```

### 2. 使用环境变量而非文件

```bash
# ✅ 推荐：使用环境变量
export OPENAI_API_KEY=sk-your-key

# ❌ 不推荐：明文存储在文件中
echo "OPENAI_API_KEY=sk-your-key" > .env
```

### 3. 限制 API Key 权限

在 OpenAI 后台：
- 设置使用限额
- 限制 IP 地址（如果支持）
- 定期轮换 API Key

### 4. 使用 .env 文件（开发环境）

```bash
# 创建 .env 文件
cp .env.example .env

# 编辑 .env 文件
nano .env

# 添加配置
CLAUDE_CODE_USE_OPENAI=1
OPENAI_API_KEY=sk-your-key-here
OPENAI_MODEL=gpt-4o
```

**注意**：确保 `.env` 文件不被提交到 Git。

## 📊 配置优先级

当多种配置方式同时存在时，优先级如下：

```
1. 命令行环境变量（最高）
   ↓
2. 项目 Profile 文件 (.openclaude-profile.json)
   ↓
3. 全局 Profile 文件 (~/.openclaude-profile.json)
   ↓
4. Shell 配置文件环境变量
   ↓
5. 系统环境变量（最低）
```

**示例**：

```bash
# 场景：多种配置同时存在

# 1. Shell 配置文件中
export OPENAI_API_KEY=sk-key-from-shell

# 2. 项目 Profile 文件中
{
  "env": {
    "OPENAI_API_KEY": "sk-key-from-profile"
  }
}

# 3. 命令行
OPENAI_API_KEY=sk-key-from-cli openclaude

# 结果：使用 sk-key-from-cli（命令行优先级最高）
```

## 🚀 快速开始指南

### 新用户推荐流程

#### Windows 用户

```powershell
# 1. 安装 OpenClaude
npm install -g @gitlawb/openclaude

# 2. 设置环境变量（永久）
[Environment]::SetEnvironmentVariable("CLAUDE_CODE_USE_OPENAI", "1", "User")
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-your-key", "User")
[Environment]::SetEnvironmentVariable("OPENAI_MODEL", "gpt-4o", "User")

# 3. 重启 PowerShell，然后启动
openclaude
```

#### macOS / Linux 用户

```bash
# 1. 安装 OpenClaude
npm install -g @gitlawb/openclaude

# 2. 添加到 Shell 配置文件
echo 'export CLAUDE_CODE_USE_OPENAI=1' >> ~/.zshrc
echo 'export OPENAI_API_KEY=sk-your-key' >> ~/.zshrc
echo 'export OPENAI_MODEL=gpt-4o' >> ~/.zshrc

# 3. 重新加载配置
source ~/.zshrc

# 4. 启动
openclaude
```

## 🔧 故障排查

### 问题 1：API Key 未生效

**症状**：
```
Error: OPENAI_API_KEY is required when CLAUDE_CODE_USE_OPENAI=1
```

**解决方案**：

```bash
# 检查环境变量是否设置
echo $OPENAI_API_KEY

# 如果为空，重新设置
export OPENAI_API_KEY=sk-your-key

# 验证
openclaude --version
```

### 问题 2：Profile 文件未被加载

**症状**：
- 创建了 `.openclaude-profile.json` 但未生效

**解决方案**：

```bash
# 确保在正确的目录
cd /path/to/your/project

# 检查文件是否存在
ls -la .openclaude-profile.json

# 验证 JSON 格式
cat .openclaude-profile.json | jq .
```

### 问题 3：API Key 格式错误

**症状**：
```
Error: Invalid OPENAI_API_KEY
```

**解决方案**：

```bash
# 检查 API Key 格式
# 正确格式：sk-xxxxxxxxxxxxxxxxxxxxxxxx

# 确保没有多余的空格或引号
export OPENAI_API_KEY=sk-proj-xxxxx  # ✅ 正确
export OPENAI_API_KEY="sk-proj-xxxxx"  # ❌ 可能有问题
export OPENAI_API_KEY='sk-proj-xxxxx'  # ❌ 可能有问题
```

## 💡 高级技巧

### 1. 多环境配置

```bash
# 开发环境
export OPENAI_API_KEY=sk-dev-key
export OPENAI_MODEL=gpt-4o

# 生产环境
export OPENAI_API_KEY=sk-prod-key
export OPENAI_MODEL=gpt-4-turbo
```

### 2. 使用不同的模型

```bash
# 快速模型
export OPENAI_MODEL=gpt-4o-mini

# 高质量模型
export OPENAI_MODEL=gpt-4o

# 代码专用
export OPENAI_MODEL=gpt-4-turbo
```

### 3. 自定义 Base URL（代理）

```bash
# 使用代理
export OPENAI_BASE_URL=https://your-proxy.com/v1

# 使用 Azure OpenAI
export OPENAI_BASE_URL=https://your-resource.openai.azure.com/
```

### 4. 切换 Provider

```bash
# 切换到 Gemini
export CLAUDE_CODE_USE_GEMINI=1
export GEMINI_API_KEY=your-gemini-key

# 切换到 Ollama
export CLAUDE_CODE_USE_OPENAI=1
export OPENAI_BASE_URL=http://localhost:11434/v1
export OPENAI_MODEL=llama3.2
```

## 📝 总结

### 推荐配置方式

| 用户类型 | 推荐方式 | 原因 |
|---------|---------|------|
| 新手用户 | Shell 配置文件 | 简单、永久、自动加载 |
| 开发者 | Profile 文件 | 灵活、可共享、多环境 |
| 临时使用 | 环境变量 | 快速、无需配置 |
| 团队协作 | Profile 文件 + Git 忽略 | 统一配置、保护密钥 |

### 配置步骤总结

1. **安装**：`npm install -g @gitlawb/openclaude`
2. **配置**：选择一种配置方式设置 API Key
3. **验证**：`openclaude --version`
4. **启动**：`openclaude`
