# OpenClaude 项目源码分析

## 📋 项目概览

**OpenClaude** 是 Claude Code 源码泄露的一个分支，核心创新是添加了 **OpenAI 兼容的 provider shim**，让用户可以使用任何 LLM（不仅仅是 Claude）来运行 Claude Code 的所有工具。

- **版本**: 0.1.7
- **技术栈**: TypeScript + React + Bun
- **核心价值**: 将 Claude Code 的强大工具能力扩展到 200+ 种 LLM
- **项目地址**: https://github.com/Gitlawb/openclaude

---

```typescript
// Anthropic → OpenAI 格式转换
function convertContentBlocks(
  content: unknown,
):
  | string
  | Array<{ type: string; text?: string; image_url?: { url: string } }> {
  if (typeof content === 'string') return content

  const parts: Array<{
    type: string
    text?: string
    image_url?: { url: string }
  }> = []

  for (const block of content) {
    switch (block.type) {
      case 'text':
        parts.push({ type: 'text', text: block.text ?? '' })
        break
      case 'image':
        // 转换图像格式
        parts.push({
          type: 'image_url',
          image_url: { url: `data:${src.media_type};base64,${src.data}` },
        })
        break
      case 'thinking':
        // 处理思考过程
        parts.push({
          type: 'text',
          text: `<thinking>${block.thinking}</thinking>`,
        })
        break
    }
  }

  return parts
}
```

### 5. 状态管理

**文件位置**: `src/state/store.ts`

采用简洁的发布-订阅模式。

#### Store 接口定义

```typescript
type Listener = () => void
type OnChange<T> = (args: { newState: T; oldState: T }) => void

export type Store<T> = {
  getState: () => T
  setState: (updater: (prev: T) => T) => void
  subscribe: (listener: Listener) => () => void
}

export function createStore<T>(
  initialState: T,
  onChange?: OnChange<T>,
): Store<T> {
  let state = initialState
  const listeners = new Set<Listener>()

  return {
    getState: () => state,

    setState: (updater: (prev: T) => T) => {
      const prev = state
      const next = updater(prev)
      if (Object.is(next, prev)) return
      state = next
      onChange?.({ newState: next, oldState: prev })
      for (const listener of listeners) listener()
    },

    subscribe: (listener: Listener) => {
      listeners.add(listener)
      return () => listeners.delete(listener)
    },
  }
}
```

#### 特点

- **不可变状态**: 所有更新通过函数式方式
- **细粒度订阅**: 支持选择性监听
- **变更追踪**: 支持 old/new state 对比
- **性能优化**: 使用 `Object.is` 避免不必要更新

```typescript
async function* streamOpenAIResponse(
  response: Response,
): AsyncGenerator<AnthropicStreamEvent> {
  const reader = response.body.getReader()
  const decoder = new TextDecoder()

  while (true) {
    const { done, value } = await reader.read()
    if (done) break

    const chunk = decoder.decode(value)
    const lines = chunk.split('\n').filter((line) => line.startsWith('data: '))

    for (const line of lines) {
      const data = JSON.parse(line.slice(6))
      yield convertToAnthropicEvent(data)
    }
  }
}
```

```typescript
export function validatePath(
  path: string,
  workingDirectory: string,
): ValidationResult {
  // 防止路径遍历攻击
  const resolved = resolve(workingDirectory, path)
  const normalized = normalize(resolved)

  if (!normalized.startsWith(workingDirectory)) {
    return {
      result: false,
      message: 'Path traversal detected',
      errorCode: 403,
    }
  }

  return { result: true }
}
```

```typescript
export function resolveCodexApiCredentials(
  env: NodeJS.ProcessEnv,
): ResolvedCodexCredentials {
  // 1. 检查环境变量
  if (env.CODEX_API_KEY) {
    return {
      apiKey: env.CODEX_API_KEY,
      accountId: env.CHATGPT_ACCOUNT_ID,
      source: 'env',
    }
  }

  // 2. 读取 auth.json
  const authPath = join(homedir(), '.codex', 'auth.json')
  if (existsSync(authPath)) {
    const auth = JSON.parse(readFileSync(authPath, 'utf8'))
    return {
      apiKey: auth.api_key,
      accountId: auth.account_id,
      authPath,
      source: 'auth.json',
    }
  }

  return { apiKey: '', source: 'none' }
}
```

OpenClaude 是一个**架构优雅、功能强大**的 AI 编程助手框架。它通过巧妙的 **API shim 层**实现了多 LLM 支持，同时保留了 Claude Code 的所有强大功能。代码质量高，模块化程度好，是学习现代 CLI 应用架构的优秀案例。

### 核心价值

1. **打破 vendor lock-in**: 不再依赖单一 LLM provider
2. **保留完整功能**: 所有工具和命令都可用
3. **易于扩展**: 插件、技能、MCP 多种扩展方式
4. **生产就绪**: 完善的错误处理、权限控制、日志系统

### 技术亮点

- 🎯 **API 兼容层**: 优雅的多 Provider 支持
- 🛡️ **安全系统**: 完善的权限和沙箱机制
- 🎨 **终端 UI**: 基于 Ink 的现代化终端界面
- 🔌 **扩展性**: 插件、技能、MCP 三位一体
- 📦 **模块化**: 高度解耦的架构设计

这是一个非常值得深入研究的开源项目！
