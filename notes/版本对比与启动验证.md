# OpenClaude 版本对比与启动验证

## 📋 版本概览

OpenClaude 提供了两个版本：

| 版本 | 类型 | 运行环境 | 主要用途 |
|------|------|---------|---------|
| **CLI 版本** | 命令行工具 | 终端 | 核心功能，直接使用 |
| **VSCode 插件版本** | VS Code 扩展 | VS Code | GUI 界面，便捷启动 |

---

## 🖥️ CLI 版本

### 什么是 CLI 版本？

CLI 版本是 OpenClaude 的**核心版本**，是一个独立的命令行工具，提供所有功能。

**特点**：
- ✅ 完整功能
- ✅ 独立运行
- ✅ 跨平台支持
- ✅ 轻量级
- ✅ 适合自动化和脚本

### CLI 版本启动验证

#### 方法 1：直接启动（推荐）

```bash
# 1. 确保已构建
bun run build

# 2. 启动 CLI
bun run start

# 或直接运行
node dist/cli.mjs
```

#### 方法 2：开发模式

```bash
# 构建并启动
bun run dev
```

#### 方法 3：使用特定 Provider

```bash
# OpenAI
bun run dev:openai

# Ollama
bun run dev:ollama

# Gemini
bun run dev:gemini
```

#### 方法 4：全局安装后启动

```bash
# 全局安装
npm install -g @gitlawb/openclaude

# 启动
openclaude
```

### CLI 版本验证步骤

```bash
# 1. 检查版本
openclaude --version
# 输出：0.1.7 (Open Claude)

# 2. 检查构建产物
ls -lh dist/cli.mjs
# 应该看到文件存在

# 3. 测试启动（需要配置 API Key）
export CLAUDE_CODE_USE_OPENAI=1
export OPENAI_API_KEY=sk-your-key
openclaude

# 4. 快速测试（无需 API Key）
openclaude --version
openclaude --help
```

### CLI 版本启动流程

```
用户执行命令
    ↓
bin/openclaude
    ↓
检查 dist/cli.mjs
    ↓
加载 cli.tsx
    ↓
环境验证
    ↓
配置加载
    ↓
启动主循环
    ↓
React UI 渲染（Ink）
    ↓
终端交互界面
```

## 🔌 VSCode 插件版本

### 什么是 VSCode 插件版本？

VSCode 插件版本是一个**VS Code 扩展**，提供图形化界面来启动和管理 OpenClaude CLI。

**特点**：
- ✅ 图形化界面
- ✅ 一键启动
- ✅ Control Center 侧边栏
- ✅ 自定义主题
- ✅ 便捷配置

**注意**：VSCode 插件版本**依赖** CLI 版本，需要先安装 CLI。

### VSCode 插件版本启动验证

#### 方法 1：开发模式启动（推荐开发调试）

```bash
# 1. 进入插件目录
cd vscode-extension/openclaude-vscode

# 2. 在 VS Code 中打开
code .

# 3. 按 F5 启动调试
# 或点击"运行和调试" → "Extension"
```

**详细步骤**：

1. **打开插件项目**：
   ```bash
   cd /Users/sijia/code/git_program/openclaude/vscode-extension/openclaude-vscode
   code .
   ```

2. **启动调试**：
   - 按 `F5` 键
   - 或点击菜单：`Run` → `Start Debugging`
   - 或点击侧边栏：`Run and Debug` → `Extension` → `Start`

3. **验证**：
   - 会打开一个新的 VS Code 窗口（扩展开发主机）
   - 左侧 Activity Bar 会出现 OpenClaude 图标
   - 点击图标可以看到 Control Center

#### 方法 2：打包安装

```bash
# 1. 进入插件目录
cd vscode-extension/openclaude-vscode

# 2. 安装依赖（如果有）
npm install

# 3. 打包插件
npm run package
# 或
npx @vscode/vsce package

# 4. 安装到 VS Code
code --install-extension openclaude-vscode-0.1.1.vsix
```

#### 方法 3：从 VSIX 文件安装

```bash
# 1. 在 VS Code 中
# 按 Cmd+Shift+P (macOS) 或 Ctrl+Shift+P (Windows/Linux)

# 2. 输入命令
Extensions: Install from VSIX...

# 3. 选择 .vsix 文件
```

### VSCode 插件验证步骤

#### 验证插件已加载

1. **打开扩展开发主机窗口**：
   - 按 `F5` 启动调试
   - 会打开一个新的 VS Code 窗口

2. **检查 Activity Bar**：
   - 左侧应该出现 OpenClaude 图标
   - 点击图标可以看到 Control Center

3. **检查命令面板**：
   - 按 `Cmd+Shift+P` (macOS) 或 `Ctrl+Shift+P` (Windows/Linux)
   - 输入 `OpenClaude`
   - 应该看到以下命令：
     - `OpenClaude: Launch in Terminal`
     - `OpenClaude: Open Control Center`
     - `OpenClaude: Open Repository`

4. **检查主题**：
   - 按 `Cmd+K Cmd+T` (macOS) 或 `Ctrl+K Ctrl+T` (Windows/Linux)
   - 应该看到 `OpenClaude Terminal Black` 主题

#### 验证插件功能

1. **测试 Control Center**：
   ```
   - 点击 Activity Bar 中的 OpenClaude 图标
   - 查看 Control Center 面板
   - 检查状态显示（runtime: available/missing）
   ```

2. **测试启动命令**：
   ```
   - 点击 "Launch OpenClaude" 按钮
   - 应该在终端中启动 OpenClaude CLI
   ```

3. **测试主题**：
   ```
   - 选择 "OpenClaude Terminal Black" 主题
   - 编辑器配色应该改变
   ```

### VSCode 插件启动流程

```
VS Code 启动
    ↓
加载插件 (extension.js)
    ↓
注册命令和视图
    ↓
创建 Control Center (Webview)
    ↓
用户点击 "Launch OpenClaude"
    ↓
检测 openclaude 命令
    ↓
创建终端并启动 CLI
    ↓
CLI 在终端中运行
```

## 🔄 两个版本的关系

### 依赖关系

```
VSCode 插件版本
    ↓
依赖 CLI 版本
    ↓
调用 openclaude 命令
    ↓
在终端中运行
```

**关键点**：
- VSCode 插件版本**不包含** CLI 功能
- 它只是一个**启动器**和**GUI 界面**
- 实际工作由 CLI 版本完成

### 功能对比

| 功能 | CLI 版本 | VSCode 插件版本 |
|------|---------|----------------|
| 核心功能 | ✅ 完整 | ❌ 无（调用 CLI） |
| 图形界面 | ❌ 终端 UI | ✅ VS Code UI |
| 一键启动 | ❌ 需要命令 | ✅ 点击按钮 |
| 配置管理 | ❌ 手动 | ✅ GUI 配置 |
| 主题支持 | ❌ 无 | ✅ 自定义主题 |
| 依赖 | ✅ 独立 | ❌ 需要 CLI |

## 🚀 完整验证流程

### 场景 1：开发环境验证

```bash
# 1. 构建 CLI 版本
cd /Users/sijia/code/git_program/openclaude
bun run build

# 2. 验证 CLI 版本
bun run start --version

# 3. 配置 API Key
export CLAUDE_CODE_USE_OPENAI=1
export OPENAI_API_KEY=sk-your-key

# 4. 启动 CLI 测试
bun run start

# 5. 测试 VSCode 插件
cd vscode-extension/openclaude-vscode
code .
# 按 F5 启动调试

# 6. 在新窗口中测试插件功能
# - 点击 OpenClaude 图标
# - 查看 Control Center
# - 点击 "Launch OpenClaude"
```

### 场景 2：生产环境验证

```bash
# 1. 全局安装 CLI
npm install -g @gitlawb/openclaude

# 2. 验证安装
openclaude --version

# 3. 配置环境变量
# macOS/Linux
echo 'export CLAUDE_CODE_USE_OPENAI=1' >> ~/.zshrc
echo 'export OPENAI_API_KEY=sk-your-key' >> ~/.zshrc
source ~/.zshrc

# Windows PowerShell
[Environment]::SetEnvironmentVariable("CLAUDE_CODE_USE_OPENAI", "1", "User")
[Environment]::SetEnvironmentVariable("OPENAI_API_KEY", "sk-your-key", "User")

# 4. 启动测试
openclaude

# 5. 安装 VSCode 插件
cd vscode-extension/openclaude-vscode
npm run package
code --install-extension openclaude-vscode-0.1.1.vsix

# 6. 重启 VS Code
# 在 Activity Bar 中找到 OpenClaude 图标
```

## 💡 使用建议

### 适合使用 CLI 版本的场景

- ✅ **终端用户**：习惯命令行操作
- ✅ **自动化脚本**：CI/CD 集成
- ✅ **远程服务器**：SSH 连接使用
- ✅ **轻量级需求**：不需要 GUI
- ✅ **快速测试**：临时使用

- ✅ **VS Code 用户**：主要使用 VS Code 开发

- ✅ **便捷启动**：一键启动 OpenClaude
- ✅ **主题统一**：使用配套主题

## 🔧 故障排查

### CLI 版本问题

#### 问题 1：找不到命令

```bash
# 症状
openclaude: command not found

# 解决方案
# 方案 1：使用完整路径
node /path/to/openclaude/dist/cli.mjs

# 方案 2：全局安装
npm install -g @gitlawb/openclaude

# 方案 3：添加到 PATH
export PATH="/path/to/openclaude/bin:$PATH"
```

#### 问题 2：构建产物不存在

```bash
# 症状
dist/cli.mjs not found

# 解决方案
bun run build
```

### VSCode 插件版本问题

#### 问题 1：插件无法启动

```bash
# 症状
点击 "Launch OpenClaude" 无反应

# 解决方案
# 1. 检查 CLI 是否安装
which openclaude

# 2. 检查配置
# VS Code 设置中搜索 "openclaude"
# 确认 launchCommand 正确

# 3. 查看输出
# VS Code → Output → OpenClaude
```

#### 问题 2：Control Center 不显示

```bash
# 症状
Activity Bar 中没有 OpenClaude 图标

# 解决方案
# 1. 重新加载窗口
# Cmd+Shift+P → "Reload Window"

# 2. 检查插件是否启用
# Extensions → 搜索 "OpenClaude" → 确认已启用

# 3. 重新安装插件
code --uninstall-extension devnull-bootloader.openclaude-vscode
code --install-extension openclaude-vscode-0.1.1.vsix
```

## 📝 总结

### 版本选择建议

| 用户类型 | 推荐版本 | 原因 |
|---------|---------|------|
| 开发者 | CLI + VSCode 插件 | 完整功能 + 便捷启动 |
| 终端用户 | CLI 版本 | 轻量级，独立运行 |
| VS Code 用户 | VSCode 插件 | 图形界面，一键启动 |
| CI/CD | CLI 版本 | 自动化，脚本友好 |

### 关键要点

1. **CLI 版本是核心**：提供所有功能
2. **VSCode 插件是启动器**：依赖 CLI 版本
3. **两者可以共存**：同时安装使用
4. **验证方法不同**：CLI 用命令，插件用 VS Code
